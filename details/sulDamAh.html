😄 안녕하세요! '자세히 보기'란 제 담당 역할에 대한 기능 설명과

프로젝트를 진행하면서 느낀점을 전달해드리기 위한 문서입니다! 📃

🧾 먼저 '주문' 페이지입니다.

사용자가 주문 페이지 내에서 기본 배송지 외 다른 배송지도 관리할 수 있도록

AJAX 통신을 활용하여 해당 기능을 구현해보았습니다.

💳 '결제' 기능입니다.

많이 부족하지만, 가장 민감한 부분이므로 다른 부분보다 더욱 신경을 쓰려고 노력했습니다.

다음과 같이 결제하는 과정에서 악의적인 사용자의 데이터 조작 행위를 방지하고자 하였습니다.

  • 결제 금액 검증

    • '바로 구매' 혹은 '장바구니 페이지'에서 주문 페이지로 넘어오는 과정에서

      사용자가 고른 상품 정보(고유번호 및 수량)를 세션에 보관해둡니다.

    • 결제를 처리하는 과정에서 먼저 포트원 API로 결제 내역 조회를 요청하여

      사용자가 실제로 결제한 금액을 응답받습니다.

    • 이후 세션에 보관해두었던 상품 정보를 토대로 계산한 금액과

      사용자가 실제로 결제한 금액이 일치하는지 검증하는 과정을 추가하였습니다.

  • 포인트 검증

    • 추가로 결제 금액이 아닌 포인트를 조작할 수도 있으므로 포인트 또한

      결제를 처리하는 과정에 사용자가 현재 보유하고 있는 포인트보다

      많은 양의 포인트를 사용하려고 시도하지는 않는지 검증하는 과정도 추가하였습니다.

❌ 다음 '주문 취소' 페이지입니다.

전체 취소 같은 경우에는 단순히 요청만 하면 돼서 큰 어려움이 없었지만,

개별 취소 같은 경우에는 고생을 좀 했던 기억이 있습니다.

API 문서를 다시 자세히 읽어보니, 'checksum'이라는 데이터의 무결성을 검증하기 위한

파라미터를 추가적으로 담아 보내야 한다는 것을 파악할 수 있었고,

그렇게 checksum 파라미터에 앞으로 취소 가능한 금액을 담아 보냄으로써

문제를 해결할 수 있었습니다.

환불 방식은 아래와 같습니다.

  • 현금 우선 환불

  • 취소할 상품이 더 존재할 경우, 배송비는 환불해주지 않고 보류

🙏 마치며

쇼핑몰이라는 주제의 프로젝트는 이전에도 몇 번 경험이 있었습니다.

하지만 항상 게시판, 상품 관리 등과 같은 기능들을 위주로 개발을 했었지,

결제 기능은 단 한 번도 구현해본 적이 없었습니다.

그래서 이번 기회에 경험해보고자 자진하여 결제 파트를 담당하게 되었습니다.

물론 테스트 연동인 만큼, 결제 시스템의 극히 일부분을 경험해봤을 뿐이고,

제 나름대로 고민하여 구현한 결제 처리 방식이 많이 미숙하다 생각하지만,

기본적인 결제 프로세스를 경험해볼 수 있었다는 것만으로도

충분히 의미 있는 경험이었습니다.
